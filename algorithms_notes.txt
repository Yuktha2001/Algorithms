# Algorithms — Study Notes

## 1. Big O / Complexity Basics
| Notation | Name | Example |
|---|---|---|
| O(1) | Constant | array index access |
| O(log n) | Logarithmic | binary search |
| O(n) | Linear | single loop |
| O(n log n) | Linearithmic | merge sort, quick sort (avg) |
| O(n²) | Quadratic | bubble sort, nested loops |
| O(2ⁿ) | Exponential | naive recursion (e.g. subsets) |

- Time complexity: how runtime grows with input size `n`.
- Space complexity: extra memory used (not counting input).
- Always analyze worst case unless stated otherwise.

## 2. Searching Algorithms
**Linear Search** — O(n)
- Check each element one by one until found.

**Binary Search** — O(log n)
- Works only on sorted arrays.
- Repeatedly halve the search range.
```java
int binarySearch(int[] arr, int target) {
    int lo = 0, hi = arr.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
```

## 3. Sorting Algorithms
| Algorithm | Best | Average | Worst | Space | Stable? |
|---|---|---|---|---|---|
| Bubble Sort | O(n) | O(n²) | O(n²) | O(1) | Yes |
| Selection Sort | O(n²) | O(n²) | O(n²) | O(1) | No |
| Insertion Sort | O(n) | O(n²) | O(n²) | O(1) | Yes |
| Merge Sort | O(n log n) | O(n log n) | O(n log n) | O(n) | Yes |
| Quick Sort | O(n log n) | O(n log n) | O(n²) | O(log n) | No |
| Heap Sort | O(n log n) | O(n log n) | O(n log n) | O(1) | No |

**Merge Sort** (divide & conquer):
```java
void mergeSort(int[] arr, int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    mergeSort(arr, l, mid);
    mergeSort(arr, mid + 1, r);
    merge(arr, l, mid, r); // combine two sorted halves
}
```

## 4. Recursion
- A function calls itself with a smaller subproblem until it hits a base case.
- Every recursive solution needs: (1) base case, (2) recursive case that moves toward it.
- Watch for stack overflow on deep recursion — consider converting to iterative/DP.

## 5. Dynamic Programming (DP)
- Used when a problem has **overlapping subproblems** and **optimal substructure**.
- Two approaches:
  - **Memoization** (top-down): recursion + cache results.
  - **Tabulation** (bottom-up): build a table iteratively.
- Classic patterns: Fibonacci, Knapsack, Longest Common Subsequence, Longest Increasing Subsequence.
```java
// Fibonacci with memoization
int fib(int n, int[] memo) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    return memo[n] = fib(n - 1, memo) + fib(n - 2, memo);
}
```

## 6. Greedy Algorithms
- Make the locally optimal choice at each step, hoping for a globally optimal result.
- Works only when the problem has the "greedy choice property."
- Examples: Activity Selection, Fractional Knapsack, Huffman Coding, Dijkstra's shortest path.

## 7. Graph Algorithms
- **BFS (Breadth-First Search)** — level-by-level traversal, uses a queue, finds shortest path in unweighted graphs.
- **DFS (Depth-First Search)** — goes as deep as possible before backtracking, uses a stack/recursion.
- **Dijkstra's Algorithm** — shortest path in weighted graphs (non-negative weights).
- **Union-Find (Disjoint Set)** — used in Kruskal's MST, cycle detection.

## 8. Tree Algorithms
- **Traversals**: Inorder, Preorder, Postorder (DFS-based), Level Order (BFS-based).
- **Binary Search Tree (BST)**: left < root < right; search/insert/delete average O(log n).
- Common problems: height of tree, diameter, lowest common ancestor (LCA), balanced tree check.

## 9. Common Problem-Solving Patterns (useful for interview prep)
- Two Pointers
- Sliding Window
- Fast & Slow Pointers (cycle detection)
- Prefix Sum
- Binary Search on Answer
- Backtracking (subsets, permutations, N-Queens)

## 10. Tips for Practice
- Always state time and space complexity when solving a problem.
- Dry-run on paper with a small example before coding.
- Group problems by pattern (not just by topic) to spot reusable techniques faster.
